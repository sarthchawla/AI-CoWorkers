# AI Scrum Master Slow Product Demo Talk Track

## Purpose

This longer demo is meant for manager sharing. It explains the flow slowly with captions and shows why the product is useful, not only that the UI works.

## Flow Covered

1. Start on the sprint plan list.
2. Show saved plans can be continued, reviewed, or cloned.
3. Explain configurable team setup:
   - team/project name
   - Jira project and board
   - Slack channel
   - sprint naming pattern
4. Start a new sprint plan.
5. Explain the guided stepper layout:
   - left workflow steps
   - center active editor
   - right sticky summary and connector status
6. Clone previous sprint context.
7. Review calendar details and working days.
8. Review the last three sprint net velocity rows.
9. Edit the manual leave request and update leave rows without running a Slack connector.
10. Save before connector actions to create an audit point.
11. Run mock Jira sprint close.
12. Pull mock Jira net velocity.
13. Review suggested net velocity calculations.
14. Override by average net velocity per developer.
15. Show the derived total sprint net velocity.
16. Finalize and save the sprint plan.
17. Return to the sprint list to show resume/review behavior.
18. Close with the product split:
    - deterministic automation for workflow, connectors, persistence, and math
    - AI for risk summaries, spillover reasoning, messy Slack interpretation, and sprint-goal recommendations

## Manager Takeaway

This is a realistic first coworker because sprint planning has a lot of deterministic workflow work that can be automated immediately. AI can then be added where judgment and summarization matter most.
